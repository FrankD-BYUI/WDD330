JavaScript: Novice to Ninja Ch2-4

Lessons Learned:

  • Comments: Single line comments use "//" and multi-line comments start with "/*" and end with "*/".
  • End each line with a semicolon to avoid issues.
  • var, let, and const are used to declare variables. var is no longer used in ES6.
  • let and const are limited in scope to the block it is delcared in.
  • Variable naming rules: "Constant and variable names can start with any upper or lower-case letter, an underscore, _ , or dollar character, $ . They can also contain numbers, but cannot start with them."
  • Template Literals use the " ` " character and allow you to include JavaScript code in within the string (just like double quotes in PHP). Example: `Hello ${ name }!`;
  • Map is a data structure that contains a list of name value pairs. Represented as Map { name => 'Value" }
  • Define a function like this: function hello(){ //body of function }
  • Define an annonymous function expression like this: const hello = function() { //body of function };
  • Define a named function expression like this: const hello = function howdy() { //body of function };
  • When you assign a function to a varable, you need the () after the variable name when you want to call it.
  • Arguments are passed in with a function call, parameters are the internal variable names inside a function.
  • Callback allows you to pass a function as an argument to another function. Remember how functions can be stored in variables? It's basically an extension of that idea.
