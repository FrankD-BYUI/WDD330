Lessons Learned:

  • Strict Mode: forces the engine enforce best practices. More structured and less forgiving than the normal 'sloppy mode'. Is enabled by adding 'use strict'; to the code.
  • Errors: throw, try, catch, finally all work pretty much as expected.
  • TTD: Test-Driven Development. Step 1 write tests, step 2 write code to pass the tests, step 3 refactor the code, step 4 test the refactored code, step 5 write more tests for new features.
  • Jest: a testing framework to simplify the testing proccess.
